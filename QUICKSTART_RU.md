# Быстрый старт

## Шаг 1: Создайте бота в Telegram

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot`
3. Придумайте имя для бота (например: "My Community Bot")
4. Придумайте username (например: "mycommunity_bot")
5. **Скопируйте токен** (выглядит как: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`)

## Шаг 2: Узнайте свой User ID

1. Откройте [@userinfobot](https://t.me/userinfobot) в Telegram
2. Отправьте `/start`
3. **Скопируйте ваш ID** (например: `123456789`)

## Шаг 3: Настройте бота

1. Скопируйте `.env.example` в `.env`:
```bash
cp .env.example .env
```

2. Откройте файл `.env` и заполните:
```bash
BOT_TOKEN=ваш_токен_от_botfather
ADMIN_IDS=ваш_user_id
CHANNEL_USERNAME=@ваш_канал  # опционально
DATABASE_PATH=bot_database.db
```

Пример:
```bash
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
ADMIN_IDS=123456789
CHANNEL_USERNAME=
DATABASE_PATH=bot_database.db
```

## Шаг 4: Установите зависимости

```bash
pip install -r requirements.txt
```

## Шаг 5: Запустите бота

```bash
python run.py
```

Вы должны увидеть:
```
INFO - Database initialized
INFO - Bot started
```

## Шаг 6: Протестируйте бота

1. Откройте вашего бота в Telegram
2. Отправьте `/start`
3. Пройдите регистрацию

## Готово!

Ваш бот запущен и работает. Теперь вы можете:

- Добавлять ресурсы через админ-панель

## Остановка бота

Нажмите `Ctrl+C` в терминале

## Возможные проблемы

### Ошибка: "BOT_TOKEN not found"
- Проверьте, что файл `.env` создан
- Проверьте, что токен скопирован правильно

### Ошибка: "Invalid token"
- Получите новый токен от BotFather
- Убедитесь, что токен без лишних пробелов

### Бот не отвечает
- Убедитесь, что бот запущен (`python run.py`)
- Проверьте логи в терминале
- Проверьте интернет соединение

## Дополнительная настройка

### Добавление нескольких администраторов

В `.env`:
```bash
ADMIN_IDS=123456789,987654321,111222333
```

### Подключение канала

1. Создайте публичный канал в Telegram
2. Добавьте username канала в `.env`:
```bash
CHANNEL_USERNAME=@mychannel
```

### Изменение пути к базе данных

В `.env`:
```bash
DATABASE_PATH=/path/to/your/database.db
```
